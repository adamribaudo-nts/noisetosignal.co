<!-- The Solution -->
<section class="py-16 px-8 text-center bg-white">
  <div class="max-w-4xl mx-auto">
    <h2 class="text-3xl font-bold mb-8">The Solution: A Governed Semantic Layer</h2>
    <p class="text-lg leading-relaxed mb-6">
      Instead of modeling data inside the BI layer, we move your business logic upstream into a universal <strong>Semantic Layer</strong>. By decoupling metrics from dashboards, we create a structured, well-governed environment where LLMs can safely and accurately query your data warehouse.
    </p>
    <p class="text-lg leading-relaxed">
      We leverage industry-leading tools like <strong><a href="https://www.getdbt.com/">dbt</a></strong> and <strong><a href="https://credibledata.com/">CredibleData</a></strong> to define your metrics as code — creating a single, machine-readable source of truth. The result: your AI gives the right answer, every time, using your organizational definitions.
    </p>
  </div>
</section>

    <div class="grid md:grid-cols-4 gap-8 leading-relaxed">
      <div>
        <div class="step-number">1</div>

      </div>
      <div>
        <div class="step-number">2</div>
        <h3 class="text-xl font-bold mb-4">Build the Semantic Layer</h3>

      </div>
      <div>
        <div class="step-number">3</div>
        <h3 class="text-xl font-bold mb-4">Connect the LLMs</h3>
        <p>We connect the semantic layer to an LLM interface. Instead of writing complex SQL against raw tables, the model uses the semantic layer's API to pull governed metrics instantly.</p>
      </div>
      <div>
        <div class="step-number">4</div>
        <h3 class="text-xl font-bold mb-4">Evaluate &amp; Iterate</h3>
